ContentService allows you to serve text content from a script, for example, by publishing the script as a web app.
When serving content through a web app, the browser URL will differ from the original script URL for security reasons.

When visiting the web app URL you will see "Hello World" in the browser. Due to security
considerations, scripts cannot directly return content to a browser. Instead, they must serve the
content from a different URL. This is why the URL in the browser for output created via this web
app will be different than the original script URL.
